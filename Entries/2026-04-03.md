## Humanity × HubSpot Integration

### How it works
- Staffing form has up to 6 call times. Each call time = one or more shifts.
- Each call time can produce up to 3 shifts: Event Staff, Bartenders, Captains (treat as separate)
- Shift start = call time value
- Shift end = event end time + 2 hours (always; Jane adjusts manually if needed)
- Shift title = Deal Name
- Shift location = Venue Address (physical address field from HubSpot)
- Open slots = headcount per role per call time

### Shift note content (what staff sees)
```
Event Name: [deal name]
Lead Cap: [lead captain name] [lead captain phone]
Venue Logistics: [venue logistics field]
```

### Trigger
- New HubSpot field: **Humanity Status** (in Staffing Numbers section)
  - `Create Shifts` → fires integration, creates all shifts
  - `Revising in Humanity` → soft signal, no action, edits happen directly in Humanity
- One-time send. After shifts exist, all edits happen in Humanity.
- If major recount change: flip status back to Create Shifts to regenerate. Staff moves are manual.

## Newco — Invoice / Contract Builder

### What it is
Replaces Bento's invoice/menu builder. Standalone web app, linked from HubSpot deals. Client never sees "Bento" or "Newco" — looks native.

### Menu sections
| Section | Source |
|---|---|
| Catering Menu (food) | Galley API |
| Bar | Galley API |
| Rentals | Internal inventory |
| Labor | Internal inventory |
| Venue fees | Internal inventory |
| Vendors | HubSpot |

### Key rules
- Price locks the moment an item is added to a contract
- Galley is source of truth for food/bar pricing — product catalog mirrors it at all times
- When event is finalized → push back to Galley as a "book"
- Vet team never touches Galley directly — they work in Newco only

### Discounts
- No per-line-item discounts
- Single discount applied at bottom of invoice
- Always defaults to last invoice in the payment schedule
- Free-text note field on discount (e.g. "complimentary chandelier")
- Discount shown as red line item on contract (e.g. -$10,000)
- Tax priority: discount taxable items first, venue fee last (venue fee has no sales tax)

### Inventory system
- Every item type has inventory, no exceptions
- Checked over date range (not globally)
- Check window: 2am–midnight daily
- Alert when any item oversold on any day
- Venue fee qty = 1. Labor = effectively unlimited. Rentals = physical count.

## Flux — Pre-Mesh Holding Space (meta)

### The idea
Flux should be the capture layer *before* mesh. Half-formed thoughts, open questions, clusters that don't have names yet. Mesh is for committed work. Flux is for everything that isn't ready to be that yet.

### What's missing
- Project-scoped context — a way to tag/group notes by project (Infinity, Flux, etc.) without formal structure
- AI summarization of a day's notes on demand
- Graduation path: Flux note gets heavy enough → one action → becomes a mesh artifact
- Backlink rendering — `[[topic]]` syntax already works as markdown, needs UI layer
- LLMService.swift is already stubbed in the codebase — needs wiring
